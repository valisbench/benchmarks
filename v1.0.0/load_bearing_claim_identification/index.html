Methodology disclosure

This wedge has a wedge-specific publication commitment. Read the rubric below before reading any score on this page.

Per-memo F1 against the ground-truth set of load-bearing claim IDs. A true positive is a claim ID the model flags as load-bearing that appears in the case's load_bearing_claim_ids after normalization (strip + lowercase). A false positive is a claim ID the model flags that is not in the ground-truth set, including invented IDs (e.g. "c99" on a memo with ten claims). A false negative is a ground-truth load-bearing ID the model misses. Refusals are recorded as a separate outcome category and excluded from the F1 calculation; per-roster refusal rates and the full seven-category outcome breakdown are reported alongside the score. Tier-3 macro precision and tier-3 macro recall are surfaced separately in the roster summary because the pre-registered hypotheses H3a and H3b test each independently. A single F1 threshold near 0.6 on tier 3 sits close to the "flag every claim" degenerate floor (recall = 1.0, precision ~ 0.40 on a tier-3 memo with 6 LB and 9 non-LB claims; F1 ~ 0.57). The precision-recall split makes that floor visible: a model that fires the precision threshold AND the recall threshold has failed both independently, not coincidentally hit one number. v1.0.0 LBCI uses synthetic generation with rule-based ground truth derived from a constructed dependency graph. The PRD originally required external human expert labelers on real memos; this requirement is overridden for v1 to ship LBCI under the parallel-build schedule for v1.0.0. v2 will commission expert labels on 80 real memos. If v2 expert-labeled F1 on any tier differs from v1 synthetic F1 by more than 0.15 (absolute) on the same model roster, v1 LBCI is deprecated and the publication updated. This trigger is pre-registered as part of the v1.0.0 anchored manifest.

Refusal policy

Refusals are scored as a separate outcome category, not as wrong answers. A refusal flag is set when the model declines to evaluate the memo; F1 is computed only over non-refused cases. Refusal rates are surfaced alongside the score so that high-refusal regimes cannot game the metric. All seven outcome categories (response, refusal, partial_response, error_timeout, error_rate_limit, error_other, malformed) are reported in the results bundle.

Failure publication commitment

Null and unfavorable results publish on the same methodology page as positive results. There is no scenario in which VALIS scores a model and withholds the finding because the result is uncomfortable. This commitment is anchored to the same Bitcoin block as the manifest. The methodology disclosure that v1.0.0 LBCI uses synthetic ground truth (with v2 expert comparison reserved, and a >0.15 absolute F1 divergence on any tier triggering deprecation) is published alongside every result.
