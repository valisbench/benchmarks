What the numbers mean

This is the finding.

Across 500 cases, Claude Opus 4.7 scored macro F1 0.40. GPT-5.5 scored 0.32. Gemini 3.1 Pro scored 0.32. Three frontier reasoning models, three independent providers, three different training runs. All three score below 0.41 on detecting what is not there.

The wedge gives the model an analysis that is internally coherent and appears complete. Somewhere in that analysis, a boundary condition that a careful reader would catch has been omitted. The cases are constructed so that the omission is detectable from the text alone -- no outside knowledge required. We ask the model to surface the omission. We score against the constructed-truth set.

Recall on the truth set sits at 0.43-0.55. The model catches roughly half of the omissions that exist. Precision is worse: when the model does flag something, it's right about 25-34% of the time. The model guesses at omissions that aren't actually missing.

This holds across providers. It holds across model architectures. It holds when we vary the wedge's case difficulty. The finding survives the spot-check panel.

What we are saying: frontier reasoning models are good at evaluating what was said. They are not good at noticing what was not said. If you are using a model to vet an argument for completeness, you should expect it to miss roughly half of the things a careful human would catch -- and to flag spurious things at a similar rate.

What would falsify this finding: any of the three frontier models hitting macro F1 above 0.60 at this case construction. None have at v1.0.0.

A natural objection is that we used a bare, one-shot prompt. A senior analyst with a year of experience using these models would not -- they would use chain-of-thought scaffolding, self-critique passes, role priming, and structured output guidance. Whether sophisticated prompting closes a 30-point gap is itself an open question. If you build or buy AI to flag what is missing from an analysis, the answer matters.

Full methodology & rubric

Scoring

Scorer: boundary_condition_omission_v1
Metric: f1

Rubric

Set-based F1 against the ground-truth boundary conditions (BCs) encoded for each case. The model's task in production: given a claim, list the things a careful reviewer should check before relying on the claim. The scorer parses the response as a list of free-text items and matches each item against the case's ground-truth BCs and distractors. Matching method (PRE-REGISTERED, locked): deterministic keyword-group matching. Each BC carries a list of "keyword_groups" -- each group is a set of concept-alternatives (synonyms/paraphrases for one BC aspect). A candidate item matches a BC iff for every group, the candidate (normalized: lowercase, punctuation collapsed to space, whitespace collapsed) contains at least one alternative from that group. The normalization is locked: no stemming, no lemmatization, no LLM-judging. Why deterministic keyword-group matching rather than embedding cosine or LLM-judge: - Reproducibility from a clean machine with no VALIS infrastructure. A third party verifying a published result must not need API access or a pinned huggingface model snapshot. Keyword-group matching needs only Python stdlib. - The ground-truth BC list and the matching rule are co-pre-registered: the keyword groups are part of the dataset under the same SHA256 the manifest commits to. Auditors can inspect both together. - Trade-off: keyword-group matching is stricter than a perfectly tuned embedding matcher would be. A model output that paraphrases the BC using novel synonyms not represented in the keyword group will miss. Keyword groups are maintained as paraphrase surfaces evolve; any change is a new manifest version. This is documented as a v1 methodological tradeoff; v2 may layer in a secondary embedding pass and report both metrics in parallel. Boundary conditions within a family share concept terms (e.g., "baseline" and "segment" both appear in multiple BCs' keyword groups). A single candidate covering multiple BC concepts can therefore satisfy multiple BCs simultaneously. This inflates recall for models whose coverage concentrates in shared concept clusters. The pre-registered choice is to accept this recall-inflation because the design hypothesis (H1) keys on recall falling below 0.4 in tier 3; recall inflation makes H1 harder to confirm, raising the falsifiability bar in the doctrine- aligned direction. Free-text model output is parsed upstream of this scorer. The harness prompts the model to return a numbered or bulleted list and parses line-by-line: one line equals one candidate item, with leading bullets/ numbers/punctuation stripped. Long prose responses are split on sentence boundaries. An empty list is treated as zero true-positives (not as a refusal). The scorer's contract begins with `Sequence[str]` candidates; the parser sits between the provider and the scorer. Per case: - tp = number of BCs covered by at least one candidate - fn = number of BCs not covered by any candidate - fp = (number of distractors flagged) + (number of unmatched candidates) - precision = tp / (tp + fp) - recall = tp / (tp + fn) - f1 = 2 * precision * recall / (precision + recall) Recall is the headline metric for the pre-registered H1 hypothesis; precision is reported alongside. Distractor-match rate is reported as a secondary diagnostic for over-flag patterns. Refusals are scored as a separate outcome category and excluded from the F1 calculation; per- roster refusal rates and the full seven-category outcome breakdown are reported alongside the score.

Refusal policy

Refusals are scored as a separate outcome category, not as wrong answers. A refusal flag is set when the model declines to evaluate the claim; F1 is computed only over non-refused cases. Refusal rates are surfaced alongside the score so that high-refusal regimes cannot game the metric. All seven outcome categories (response, refusal, partial_response, error_timeout, error_rate_limit, error_other, malformed) are reported in the results bundle.

Failure publication commitment

Null and unfavorable results publish on the same methodology page as positive results. There is no scenario in which VALIS scores a model and withholds the finding because the result is uncomfortable. This commitment is anchored to the same Bitcoin block as the manifest.

Pre-registered hypotheses

  1. H1: Set recall on ground-truth boundary conditions will fall below 0.4 on at least one frontier model in tier 3 (claim has implicit assumptions that are easy to miss; 7-8 BCs and 5 topically-overlapping distractors).
